# mmWave-SLAM-RaspRover

基于 Waveshare RaspRover 无人车平台的毫米波 + 激光雷达 SLAM 实验系统。

本项目目标是构建一个可扩展的无人探测车研究平台，用于验证：

- 激光雷达 SLAM 建图与自主导航
- 毫米波雷达作为补充传感器的实际价值
- Raspberry Pi + ROS2 架构在移动平台中的可行性
- 独立无线操作终端的人机交互方式


---

## 系统组成

系统由四个逻辑节点构成：

- RaspRover 小车（内部 ESP32 控制底盘，黑盒）
- Raspberry Pi 5（Ubuntu + ROS2，核心计算）
- ESP32-P4（无线操作终端 + 显示）
- PC Ubuntu（开发 / 可视化）

RaspRover 自带 ESP32 负责电机与编码器控制，本项目不修改其固件，仅通过上位接口控制小车运动。

---

## 各模块职责

### RaspRover 小车（底盘系统 / 黑盒）

- 内置 ESP32 控制电机与编码器
- 提供 SDK / 通信接口
- 本项目不开发其底层控制逻辑

定位：

移动平台执行单元


---

### Raspberry Pi 5 + Ubuntu（核心计算节点）

负责：

- 毫米波雷达采集
- 激光雷达数据接入
- ROS2
- SLAM
- 自主导航逻辑
- 向 RaspRover 下发运动指令
- 向 PCubuntu 提供状态数据
- 接收EPS32-P4的指令

特点：

- 系统核心
- 运行 Ubuntu + ROS2
- 不做人机界面
- 不做点云可视化

定位：

感知 + 决策中心


---

### ESP32-P4（无线操作终端）

负责：

- 电容屏 UI
- 遥控输入
- 状态显示
- 地图 / 可视化数据简化显示（未来）

特点：

- 独立无线节点
- 不参与底盘控制
- 不运行 ROS
- 不直接驱动电机
- 可与 Raspberry Pi 和 PC 通信

定位：

人机交互终端


---

### PC Ubuntu（开发 / 可视化）

负责：

- RViz
- 点云显示
- 地图查看
- ROS2 调试
- 参数调节
- 日志分析

特点：

- 工程调试用途
- 不参与实时控制

定位：

工程工作站


---

## 设计原则

- 不修改 RaspRover 底盘固件
- Raspberry Pi 是系统唯一决策节点
- ESP32-P4 只做人机交互
- PC 仅用于开发与观察
- 所有通信使用清晰文本 / 网络协议
- 架构保持松耦合
- 各节点可独立调试
