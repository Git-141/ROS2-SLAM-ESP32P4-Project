# 📄 GUI 主页设计说明（第二阶段系统）

## 1. 设计目标

本界面用于第二阶段系统的人机交互控制终端，运行于 **ESP32-P4 + 10寸触控屏**。

设计目标：

* 提供系统运行状态总览
* 作为各功能模块入口
* 不展示底层原始数据（如点云、单点距离）
* 保持界面简洁、响应快速

---

## 2. 设计原则

### 2.1 分层原则

界面只负责显示：

* ✔ 系统级结果（如状态、位姿）
* ✔ 设备状态
* ✔ 功能入口

不负责：

* ❌ 点云筛选
* ❌ 距离计算
* ❌ 数据滤波

---

### 2.2 信息优先级

| 层级 | 内容   | 说明        |
| -- | ---- | --------- |
| 高  | 系统状态 | 是否正常运行    |
| 中  | 位姿信息 | (x, y, θ) |
| 中  | 设备状态 | 雷达是否正常    |
| 低  | 功能入口 | 用户操作入口    |

---

### 2.3 交互原则

* 主页不提供复杂操作
* 仅作为功能入口与状态展示
* 所有详细功能进入卡片页面
* 保留底部导航（返回 / 首页）

---

# 📄 网络菜单设计说明（第二阶段系统）

## 1. 设计目标

本模块用于管理系统无线通信功能，运行于 **ESP32-P4 + 10寸触控屏**，并与树莓派协同工作。

设计目标：

* 提供 ESP32 当前网络状态展示
* 提供 WiFi 选择与切换功能
* 检测 ESP32 与树莓派通信状态
* 在热点模式下，实现对树莓派网络的远程配置
* 提供基础输入能力（软键盘）

---

## 2. 设计原则

### 2.1 分层原则

ESP32-P4 负责：

* ✔ 用户交互（触摸、输入）
* ✔ 网络状态显示
* ✔ 向树莓派发送控制指令

树莓派负责：

* ✔ 实际 WiFi 扫描
* ✔ 网络连接执行
* ✔ 网络模式切换

说明：

* ESP32 不直接控制树莓派网络
* 所有配置通过通信协议下发

---

### 2.2 信息优先级

| 层级 | 内容         | 说明         |
| ---- | ------------ | ------------ |
| 高   | 网络连接状态 | 是否在线     |
| 中   | IP / SSID    | 当前网络信息 |
| 中   | Pi通信状态   | 是否可交互   |
| 低   | 操作入口     | 配置与控制   |

---

### 2.3 交互原则

* 网络状态优先展示
* 操作分步进行（扫描 → 选择 → 输入 → 连接）
* 输入操作独立页面（软键盘）
* 避免在主页进行复杂操作

---

## 6. 网络切换逻辑说明（重要）

在热点模式下：

1. ESP32 连接树莓派热点
2. 用户输入目标 WiFi 信息
3. ESP32 发送配置给树莓派
4. 树莓派尝试连接新网络
5. 成功后返回新 IP
6. ESP32 自动切换至同一网络
7. 重新建立通信

| Pi失联      | 提示通信异常 |
| ESP32未重连 | 提示手动连接 |

控制菜单设计说明（第二阶段系统 · 完整版）
1. 设计目标

本模块用于系统手动控制终端，运行于 ESP32-P4 + 10寸触控屏，用于对小车与舵机云台进行实时操作控制。

设计目标：

提供车体移动控制
提供舵机方向控制
提供安全控制机制（刹车 / 急停 / 锁定）
提供基础状态反馈
保持操作直观、响应快速
2. 设计原则
2.1 功能分离原则

控制对象分为两类：

✔ 车体运动（底盘）
✔ 舵机运动（云台）

分别独立控制，避免混用。

2.2 直观操作原则

采用双摇杆布局：

左摇杆 → 控制车体
右摇杆 → 控制舵机

无需切换控制模式。

2.3 安全优先原则

所有控制必须受以下机制约束：

控制使能（锁定/启用）
刹车
急停
2.4 输入稳定性原则

为避免触控抖动：

设置摇杆死区（Dead Zone）
限制最大变化速率
支持速度倍率调节
3. 页面结构示意
┌──────────────────────────────┐
│ 控制                         │
├──────────────────────────────┤
│ 模式：手动    控制：已锁定     │
│ 速度：0.32 m/s               │
│ 舵机：X=15°  Y=-10°          │
│ Pi通信：OK                   │
├──────────────────────────────┤
│                              │
│   [ 左摇杆 ]   [ 右摇杆 ]     │
│    车体移动      舵机控制      │
│                              │
├──────────────────────────────┤
│ [ 启用 ] [ 停止 ] [ 刹车 ]     │
│ [ 回中 ] [ 慢速 ] [ 急停 ]     │
│                              │
│ [ 返回 ] [ 首页 ]             │
└──────────────────────────────┘
4. 功能说明
4.1 左摇杆（车体控制）

控制小车运动：

上：前进
下：后退
左：左转
右：右转

建议映射为差速控制：

left_speed  = y + x
right_speed = y - x
4.2 右摇杆（舵机控制）

控制云台方向：

左右：水平（Yaw）
上下：俯仰（Pitch）
4.3 控制使能（启用 / 锁定）
[ 启用 ] / [ 锁定 ]

功能：

锁定：所有控制输入无效
启用：允许摇杆与按钮操作

默认状态：

进入页面时为 锁定
4.4 停止

功能：

将速度归零
自然减速停止
4.5 刹车（Brake）

功能：

立即停止电机输出
强制制动

优先级：

高于摇杆输入
4.6 急停（Emergency Stop）

功能：

停止所有控制输出
锁定系统
必须手动解除

说明：

系统最高优先级控制
4.7 舵机回中

功能：

Yaw = 0°
Pitch = 0°

建议：

平滑回中（避免瞬间跳变）
4.8 速度模式
[ 慢速 ] / [ 正常 ] / [ 快速 ]

作用：

控制最大速度输出比例
提升操作精度
5. 输入处理逻辑
5.1 摇杆死区（Dead Zone）
|x| < 0.05 → 0
|y| < 0.05 → 0

作用：

防止抖动
防止误触
5.2 输出限幅
-1.0 ≤ x, y ≤ 1.0
5.3 平滑处理

建议：

限制相邻帧变化量
防止突变
6. 控制数据结构（通信协议）

ESP32 向树莓派发送控制指令：

CMD:CONTROL
ENABLE=1
VX=0.32
WZ=0.15
YAW=12
PITCH=-8
MODE=MANUAL

说明：

字段	含义
ENABLE	控制使能
VX	前进速度
WZ	转向
YAW	水平角
PITCH	俯仰角
MODE	控制模式
7. 控制流程
正常控制流程
用户启用控制
操作双摇杆
ESP32 生成控制数据
发送至树莓派
树莓派执行控制
刹车流程
用户点击刹车
立即发送停止指令
忽略摇杆输入
急停流程
用户点击急停
系统锁定
所有控制停止
需手动解除
8. 兼容性设计

若触摸屏不支持多点触控：

可降级为：

单摇杆 + 舵机按钮
单摇杆 + 控制对象切换

## 状态菜单设计说明（第二阶段系统 · 完整版）
1. 设计目标

本模块用于系统运行状态监控与信息展示，运行于 ESP32-P4 + 10寸触控屏，用于对系统整体运行情况进行可视化管理。

设计目标：

提供系统运行状态总览
显示 ESP32-P4 与树莓派关键状态信息
显示系统资源占用情况
显示通信与服务状态
提供告警与日志功能

2.1 分层展示原则
