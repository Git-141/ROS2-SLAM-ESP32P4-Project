# Pi 4B 触控 GUI 设计

本文档记录当前 Raspberry Pi 4B 触控终端的目标界面。

设计参考来自 `image2` 生成的遥控器概览图：深色横屏控制台、卡片式布局、首页总览、二级菜单概览和详细功能页面。

## 1. 可实现性判断

这套界面在 Raspberry Pi 4B 上可以实现，而且比 ESP32-P4 路线更适合。

原因：

- Pi 4B 可以运行 Chromium kiosk，全屏显示 Web UI。
- 10.1 寸横屏空间足够承载多卡片控制台。
- 中文字体、图标、动画、地图画布、日志列表都可以用常规 Web 技术实现。
- 摄像头画面、地图摘要、雷达扫描图和状态图表可以先用模拟数据占位，后续再接 Pi 5 的真实数据。
- 不再需要处理 MCU 上的 LVGL 字库、FreeType 栈、SD 卡位图字库等限制。

第一版建议做成静态 / 半动态 Web UI，先把布局和触控体验跑通，再接真实数据。

## 2. 总体风格

目标风格：

- 深色背景。
- 蓝色作为主强调色。
- 青色用于在线、连接、选中和速度等实时状态。
- 绿色表示正常。
- 黄色表示警告。
- 红色表示错误或危险。
- 使用细边框、低亮度卡片和紧凑信息排布。

界面气质：

- 工程控制台。
- 车载遥控终端。
- 信息密度高，但层次清晰。
- 所有主要操作适配触控。

## 3. 屏幕与布局基准

目标屏幕：

- 设备：Raspberry Pi 4B + Waveshare 10.1 寸触控屏。
- 优先方向：横屏。
- 目标分辨率：以 `1280 x 800` 或同等级宽屏布局为基准。

布局原则：

- 首页不做欢迎页，开机直接进入控制总览。
- 顶部固定状态栏。
- 底部固定一级导航。
- 中间区域使用卡片网格。
- 关键控制区和相机画面优先级最高。
- 所有按钮尺寸要适合手指点击。

## 4. 一级菜单

底部一级导航保留 7 个入口：

| 菜单 | 用途 |
| --- | --- |
| 首页 | 总览、遥控、相机、地图、雷达、日志 |
| 地图 | SLAM / 地图 / 位姿详细页面 |
| 控制 | 运动控制、云台控制、速度模式 |
| 状态 | Pi 5、传感器、网络、电池、服务状态 |
| 参数 | 运动参数、传感器参数、导航参数、网络参数 |
| 日志 | 系统日志、告警、筛选、导出 |
| 设置 | 界面、网络、系统、设备管理、关于 |

一级导航始终可见，当前页高亮。

## 5. 首页总览

首页是主工作界面，参考图中左上大屏布局。

### 5.1 顶部状态栏

显示内容：

- 终端名称：`UGV 遥控终端`。
- WiFi 连接状态。
- 信号强度。
- 当前时间。
- 系统负载或 CPU 占用。
- 电池 / 电源状态。
- 设置入口。

### 5.2 左侧运动控制卡片

内容：

- 虚拟方向摇杆。
- 前进、后退、左转、右转方向提示。
- 模式切换：低速 / 中速 / 高速。
- 当前速度显示。

第一版可以只做触控 UI 和模拟输出，不直接接底盘控制。

### 5.3 相机画面卡片

内容：

- 前置摄像头画面。
- 全屏按钮。
- 画面来源状态。
- 后续可加入云台方向或目标框。

第一版可以使用占位图或测试视频流。

### 5.4 快捷菜单卡片

提供 6 个快捷入口：

- 地图。
- 控制。
- 状态。
- 参数。
- 日志。
- 设置。

卡片使用图标加短文字，点击后进入对应二级页面。

### 5.5 系统状态卡片

显示：

- Raspberry Pi 5 状态。
- 电池电压。
- 电池余量。
- 电机状态。
- 激光雷达状态。
- WiFi 信号。

状态只显示结论，不显示复杂原始数据。

### 5.6 SLAM 地图卡片

显示：

- 小地图。
- 当前车体位置。
- 朝向箭头。
- 2D / 3D 切换入口。
- 地图更新时间状态。

首页只展示摘要，复杂调试仍交给地图详细页或 PC RViz。

### 5.7 雷达扫描卡片

显示：

- 圆形雷达视图。
- 角度刻度。
- 障碍点颜色。
- 距离单位。

第一版可以用模拟点云动画占位。

### 5.8 日志信息卡片

显示最近事件：

- INFO。
- WARN。
- ERROR。
- 时间戳。
- 简短文本。

首页只显示最近几条，完整内容进入日志页。

## 6. 二级菜单概览

二级菜单概览用于快速看到每个模块的缩略状态。

参考图右上区域，采用 2 x 3 卡片：

- 地图。
- 控制。
- 状态。
- 参数。
- 日志。
- 设置。

每张卡片展示该模块最关键的信息，不做复杂操作。

适用场景：

- 从首页点击“菜单”或某个快捷入口前，先看到模块总览。
- 后续也可以作为平板式 launcher。

## 7. 详细页面

### 7.1 地图页面

内容：

- 2D 地图。
- 3D 地图入口。
- 路径规划入口。
- 地图层开关。
- 显示轨迹点、障碍物、当前位置。
- 视角重置和保存地图按钮。

### 7.2 控制页面

内容：

- 手动控制。
- 自动导航。
- 功能控制。
- 运动摇杆。
- 云台 / 灯光 / 一键回中。
- 速度限制。
- 转向灵敏度。
- 低速 / 中速 / 高速模式。

安全要求：

- 默认进入页面时不直接使能运动。
- 急停和刹车必须有明显视觉优先级。
- 控制输出必须等待后续安全策略确认。

### 7.3 状态页面

内容：

- Pi 5 CPU、内存、温度、运行时间。
- 风扇转速。
- 传感器状态。
- 网络状态。
- 电池状态。
- 服务状态。

页面重点是快速判断系统是否健康。

### 7.4 参数页面

内容：

- 运动参数。
- 传感器参数。
- 导航参数。
- 网络参数。
- 滑条、数字输入和保存按钮。

第一版只做 UI 占位，实际写入配置需等参数体系确定。

### 7.5 日志页面

内容：

- 日志列表。
- INFO / WARN / ERROR 筛选。
- 清空日志。
- 导出日志。

日志行需要清晰区分等级颜色。

### 7.6 设置页面

内容：

- 界面设置。
- 网络设置。
- 系统设置。
- 设备管理。
- 关于我们。

界面设置可先包含：

- 深色 / 浅色模式。
- 界面缩放。
- 语言设置。
- 开机自启动。
- 自动锁屏时间。

## 8. 第一版实现边界

第一版目标：

- 做出和概览图接近的静态布局。
- 支持触控点击和页面切换。
- 使用模拟数据驱动状态、地图、雷达、日志。
- 相机区域先使用占位图或测试流。
- 不接真实底盘控制。
- 不固定 Pi 5 通信接口。

第一版不做：

- 真正下发运动控制。
- 真实 SLAM 地图订阅。
- 真实雷达点云显示。
- 参数持久化。
- 日志导出。

## 9. 推荐实现方式

推荐使用 Web UI：

- Pi 4B 启动 Chromium kiosk。
- 前端负责所有界面绘制。
- 图标使用 Web 图标库。
- 地图和雷达区域使用 Canvas 或 SVG。
- 数据层先用 mock，再替换为真实服务。

后续目录建议：

```text
touch_terminal/
  web/
    src/
    public/
    package.json
```

## 10. 当前结论

这张概览图可以作为 Pi 4B 触控 GUI 的主视觉目标。

当前应把它定为：

- 首页总览目标。
- 二级菜单结构参考。
- 详细页面信息架构参考。

后续开发时优先实现首页、控制页、状态页、日志页，再补地图和参数页面。
