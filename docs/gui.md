# 📄 GUI 主页设计说明（第二阶段系统）

## 1. 设计目标

本界面用于第二阶段系统的人机交互控制终端，运行于 **ESP32-P4 + 10寸触控屏**。

设计目标：

* 提供系统运行状态总览
* 作为各功能模块入口
* 不展示底层原始数据（如点云、单点距离）
* 保持界面简洁、响应快速

---

## 2. 设计原则

### 2.1 分层原则

界面只负责显示：

* ✔ 系统级结果（如状态、位姿）
* ✔ 设备状态
* ✔ 功能入口

不负责：

* ❌ 点云筛选
* ❌ 距离计算
* ❌ 数据滤波

---

### 2.2 信息优先级

| 层级 | 内容   | 说明        |
| -- | ---- | --------- |
| 高  | 系统状态 | 是否正常运行    |
| 中  | 位姿信息 | (x, y, θ) |
| 中  | 设备状态 | 雷达是否正常    |
| 低  | 功能入口 | 用户操作入口    |

---

### 2.3 交互原则

* 主页不提供复杂操作
* 仅作为功能入口与状态展示
* 所有详细功能进入卡片页面
* 保留底部导航（返回 / 首页）

---

---

## 4. 各区域功能说明

### 4.1 状态栏

显示连接状态：

* WiFi：是否联网
* Pi：是否与树莓派通信正常

---

### 4.2 系统信息区

显示系统核心信息：

* **位置 (x, y, θ)**
* **系统状态（NORMAL）**
* **设备状态**

  * 毫米波雷达（MMW）
  * 激光雷达（Lidar）

说明：

* 仅显示“状态”，不展示原始数据
* 使用简洁标识（如 ●）表示状态

---

### 4.3 卡片入口区

提供主要功能入口：

| 卡片   | 功能       |
| ---- | -------- |
| 网络   | 通信相关信息   |
| 控制   | 操作控制     |
| 毫米波  | 毫米波雷达信息  |
| 激光   | 激光雷达信息   |
| SLAM | SLAM相关功能 |
| 状态   | 系统运行状态   |

特点：

* 采用 2×3 布局
* 适配触控操作
* 为主要操作区域

---

### 4.4 底部导航栏

```text
← 返回        🏠 首页
```

功能：

* 返回：回到上一级页面
* 首页：回到主页

说明：

* 统一导航逻辑
* 保持操作一致性
