# 📄 GUI 主页设计说明（第二阶段系统）

## 1. 设计目标

本界面用于第二阶段系统的人机交互控制终端，运行于 **ESP32-P4 + 10寸触控屏**。

设计目标：

* 提供系统运行状态总览
* 作为各功能模块入口
* 不展示底层原始数据（如点云、单点距离）
* 保持界面简洁、响应快速

---

## 2. 设计原则

### 2.1 分层原则

界面只负责显示：

* ✔ 系统级结果（如状态、位姿）
* ✔ 设备状态
* ✔ 功能入口

不负责：

* ❌ 点云筛选
* ❌ 距离计算
* ❌ 数据滤波

---

### 2.2 信息优先级

| 层级 | 内容   | 说明        |
| -- | ---- | --------- |
| 高  | 系统状态 | 是否正常运行    |
| 中  | 位姿信息 | (x, y, θ) |
| 中  | 设备状态 | 雷达是否正常    |
| 低  | 功能入口 | 用户操作入口    |

---

### 2.3 交互原则

* 主页不提供复杂操作
* 仅作为功能入口与状态展示
* 所有详细功能进入卡片页面
* 保留底部导航（返回 / 首页）

---

## 3. 主页结构示意

```text
┌──────────────────────────────┐

├──────────────────────────────┤

├──────────────────────────────┤

│ [ 网络 ]      [ 控制 ]        │
│                              │
│ [ 毫米波 ]    [ 激光 ]        │
│                              │
│ [ SLAM ]      [ 状态 ]        │
│                              │
├──────────────────────────────┤
│ ← 返回        🏠 首页          │
└──────────────────────────────┘
```

---

## 4. 各区域功能说明

### 4.1 状态栏

显示连接状态：

* WiFi：是否联网
* Pi：是否与树莓派通信正常

---

### 4.2 系统信息区

显示系统核心信息：

* **位置 (x, y, θ)**
* **系统状态（NORMAL）**
* **设备状态**

  * 毫米波雷达（MMW）
  * 激光雷达（Lidar）

说明：

* 仅显示“状态”，不展示原始数据
* 使用简洁标识（如 ●）表示状态

---

### 4.3 卡片入口区

提供主要功能入口：

| 卡片   | 功能       |
| ---- | -------- |
| 网络   | 通信相关信息   |
| 控制   | 操作控制     |
| 毫米波  | 毫米波雷达信息  |
| 激光   | 激光雷达信息   |
| SLAM | SLAM相关功能 |
| 状态   | 系统运行状态   |

特点：

* 采用 2×3 布局
* 适配触控操作
* 为主要操作区域

---

### 4.4 底部导航栏

```text
← 返回        🏠 首页
```

功能：

* 返回：回到上一级页面
* 首页：回到主页

说明：

* 统一导航逻辑
* 保持操作一致性

# 📄 网络菜单设计说明（第二阶段系统）

## 1. 设计目标

本模块用于管理系统无线通信功能，运行于 **ESP32-P4 + 10寸触控屏**，并与树莓派协同工作。

设计目标：

* 提供 ESP32 当前网络状态展示
* 提供 WiFi 选择与切换功能
* 检测 ESP32 与树莓派通信状态
* 在热点模式下，实现对树莓派网络的远程配置
* 提供基础输入能力（软键盘）

---

## 2. 设计原则

### 2.1 分层原则

ESP32-P4 负责：

* ✔ 用户交互（触摸、输入）
* ✔ 网络状态显示
* ✔ 向树莓派发送控制指令

树莓派负责：

* ✔ 实际 WiFi 扫描
* ✔ 网络连接执行
* ✔ 网络模式切换

说明：

* ESP32 不直接控制树莓派网络
* 所有配置通过通信协议下发

---

### 2.2 信息优先级

| 层级 | 内容         | 说明         |
| ---- | ------------ | ------------ |
| 高   | 网络连接状态 | 是否在线     |
| 中   | IP / SSID    | 当前网络信息 |
| 中   | Pi通信状态   | 是否可交互   |
| 低   | 操作入口     | 配置与控制   |

---

### 2.3 交互原则

* 网络状态优先展示
* 操作分步进行（扫描 → 选择 → 输入 → 连接）
* 输入操作独立页面（软键盘）
* 避免在主页进行复杂操作

---

说明：

* 使用应用层通信（非仅WiFi状态）

## 5. 软键盘设计（关键部分）

### 5.1 键盘布局

```text
┌──────────────────────────────┐
│ q w e r t y u i o p          │
│ a s d f g h j k l            │
│ z x c v b n m                │
│                              │
│ 1 2 3 4 5 6 7 8 9 0          │
│                              │
│ [切换] [空格] [删除] [确定]   │
└──────────────────────────────┘
```

---

### 5.2 功能说明

| 按键      | 功能               |
| --------- | ------------------ |
| 字母/数字 | 输入字符           |
| 删除      | 删除一位           |
| 空格      | 输入空格           |
| 切换      | 字母 / 数字 / 符号 |
| 确认      | 提交输入           |

---

### 5.3 输入逻辑

* 支持隐藏密码显示（`******`）
* 可切换显示明文
* 输入完成后发送至树莓派：

```text
CMD:PI_WIFI_CONNECT
SSID=Lab_WiFi
PASS=12345678
```

---

## 6. 网络切换逻辑说明（重要）

在热点模式下：

1. ESP32 连接树莓派热点
2. 用户输入目标 WiFi 信息
3. ESP32 发送配置给树莓派
4. 树莓派尝试连接新网络
5. 成功后返回新 IP
6. ESP32 自动切换至同一网络
7. 重新建立通信

---

## 7. 异常处理

| 情况        | 处理方式     |
| ----------- | ------------ |
| 密码错误    | Pi 返回失败  |
| 连接失败    | 自动恢复热点 |
| Pi失联      | 提示通信异常 |
| ESP32未重连 | 提示手动连接 |
