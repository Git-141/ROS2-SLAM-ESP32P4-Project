# Raspberry Pi 4B 触控终端设计

本文档替代旧的 ESP32-P4 无线操作终端路线。

## 1. 目标

使用 `Raspberry Pi 4B + Waveshare 10.1 寸 DSI 触控屏` 作为系统操作终端。

它负责：

- 显示系统运行状态。
- 提供手动遥控入口。
- 管理网络与连接状态。
- 显示传感器、SLAM、日志等摘要信息。
- 与 Raspberry Pi 5 通信，但不直接驱动底盘。

## 2. 为什么改用 Pi 4B

相比 ESP32-P4 路线，Pi 4B 的优势是：

- Linux 图形栈成熟，UI 开发更快。
- 中文、日文、图标、地图显示不再受 MCU 字库限制。
- 可直接运行浏览器、Qt、Python、Node.js 等常规工具。
- 网络、文件传输和调试更简单。
- 同款 10.1 寸屏更适合承载复杂页面。

需要注意：

- Pi 4B 功耗高于 ESP32-P4。
- 启动时间更长。
- 系统镜像、桌面环境和服务管理需要维护。

## 3. 推荐 UI 路线

首选：本地 Web UI。

建议方式：

- Pi 4B 本机运行 Chromium kiosk 全屏模式。
- UI 服务可运行在 Pi 4B 本机，也可以由 Pi 5 提供。
- Pi 4B 通过 HTTP / WebSocket 访问 Pi 5 的状态和控制接口。

推荐技术栈：

- 前端：普通 HTML/CSS/JavaScript，或后续再上 React/Vue。
- 后端：Python FastAPI / Flask，或 Node.js。
- 实时状态：WebSocket。
- 配置与文件传输：HTTP REST。

## 4. 页面规划

GUI 目标参考 `image2` 生成的遥控器概览图，主方向是横屏深色控制台。

详细 GUI 设计见：

- `docs/gui.md`

第一版页面：

- 首页：运动控制、相机画面、快捷菜单、系统状态、SLAM 地图、雷达扫描、日志摘要。
- 控制页：手动遥控、自动导航、功能控制、速度模式、云台控制。
- 状态页：Pi 5、传感器、网络、电池、服务状态。
- 参数页：运动参数、传感器参数、导航参数、网络参数。
- 日志页：事件、告警、筛选、清空、导出入口。
- 设置页：界面、网络、系统、设备管理、关于。

第二版页面：

- 地图详细页。
- 雷达详细页。
- 摄像头全屏页。
- 简化任务规划页。

## 5. 屏幕迁移注意事项

同款屏幕在 ESP32-P4 上的经验可以保留为参考，但不能直接照搬。

需要重新确认：

- Pi 4B 是否支持该屏幕的 DSI 接口和分辨率。
- 触摸是否通过 I2C / USB / HID 正常识别。
- 横屏 / 竖屏方向。
- 触摸坐标是否需要旋转或校准。
- 开机后是否能自动进入 kiosk UI。

旧 ESP32-P4 的这些内容不再作为实现约束：

- LVGL。
- ESP-IDF。
- SD 卡位图字库。
- FreeType 栈和字形缓存调优。
- ESP32-C6 hosted WiFi。

## 6. 和 Pi 5 的控制关系

Pi 4B 只发送“用户意图”，Pi 5 负责最终安全裁决。

Pi 5 收到后需要：

- 检查急停状态。
- 检查通信超时。
- 限幅和限速。
- 转换为底盘 SDK 指令。
- 回传实际执行状态。
