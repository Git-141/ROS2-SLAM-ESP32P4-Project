# UGV 遥控终端 UI 开发提示词：无假数据版

## 1. 项目目标

开发一个运行在 **Raspberry Pi 4B + 10.1 寸电容触摸屏** 上的 UGV 遥控终端界面。

系统用于：

```text
车体运动控制
云台 / 雷达方向控制
摄像头画面显示
SLAM 地图显示
传感器状态显示
树莓派 4B / 树莓派 5 / Ubuntu PC 状态监视
网络连接管理
日志查看
参数设置
```

项目优先使用：

```text
后端：Python + FastAPI / Flask
前端：HTML + CSS + JavaScript
实时通信：WebSocket
运行平台：Raspberry Pi 4B
显示设备：10.1 寸横屏电容触摸屏
目标分辨率：1280×720 或 1366×768
```

重要原则：

```text
不要生成假数据
不要写随机数据
不要用 setInterval 随机刷新状态
不要为了展示效果伪造 CPU、温度、电池、SLAM、日志、雷达、地图数据
所有动态数据都必须来自后端接口或 WebSocket
当前没有真实数据时，前端显示“未连接”“等待数据”“暂无数据”
```

# 3. 数据原则

## 3.1 禁止假数据

Codex 必须遵守：

```text

禁止生成随机系统状态
禁止生成随机日志
禁止生成随机 SLAM 地图
```

允许使用：

```text

暂无数据
未连接
等待后端数据
接口返回 null
接口返回空数组
WebSocket 暂无消息
```

# 5. 后端接口模块

## 5.1 state_manager.py

作用：

```text
统一保存当前系统状态
接收各模块真实返回值
向 WebSocket 提供当前状态快照
```

状态字段：

```text
Pi4B 状态
Pi5 状态
PC 状态
网络状态
车辆状态
云台状态
传感器状态
SLAM 状态
日志列表
```

没有数据时：

```json
{
  "pi4b": null,
  "pi5": null,
  "pc": null,
  "vehicle": null,
  "gimbal": null,
  "slam": null,
  "logs": []
}
```

---

## 5.2 websocket_server.py

作用：

```text
建立前后端实时通信
向前端推送真实状态变化
接收前端控制指令
```

注意：

```text
WebSocket 不主动生成假数据。
只有后端状态变化时才推送。
如果暂时没有真实数据，则只推送连接状态或空状态。
```

消息格式：

```json
{
  "type": "status_update",
  "data": null,
  "status": "waiting"
}
```

```json
{
  "type": "vehicle_state",
  "data": null,
  "status": "not_connected"
}
```

```json
{
  "type": "log",
  "data": [],
  "status": "empty"
}
```

---

## 5.3 system_monitor.py

作用：

```text
读取 Pi4B 本机真实系统状态
```

可以实现真实读取：

```text
CPU 使用率
内存使用
温度
磁盘使用
当前时间
```

推荐库：

```text
psutil
subprocess
```

注意：

```text
Pi4B 本机状态可以读取真实数据。
Pi5 和 PC 状态不能伪造。
Pi5 / PC 未接入前显示未连接。
```

---

## 5.4 network_manager.py

作用：

```text
读取和管理网络状态
```

预留功能：

```text
读取当前 SSID
读取 IP 地址
读取 Wi-Fi 信号强度
扫描 Wi-Fi
连接 Wi-Fi
切换热点模式
```

可使用：

```text
nmcli
ip addr
iwconfig
NetworkManager
```

初期要求：

```text
可以只读取 Pi4B 当前网络真实状态。
不要伪造 Pi5 / PC 网络状态。
未接入时显示未连接。
```

---

## 5.5 control_bridge.py

作用：

```text
统一处理车辆控制和云台控制指令
```

接口：

```python
def send_vehicle_command(vx: float, wz: float):
    pass

def send_gimbal_command(pan: float, tilt: float):
    pass

def send_emergency_stop():
    pass
```

当前阶段：

```text
可以记录收到的指令
可以写入日志
不要真正驱动硬件
不要伪造执行成功
返回 pending / not_connected
```

返回示例：

```json
{
  "ok": false,
  "status": "not_connected",
  "message": "vehicle control interface is not connected"
}
```

---

## 5.6 ros_bridge.py

作用：

```text
预留 ROS2 接口
```

未来对接：

```text
/cmd_vel
/odom
/map
/scan
/tf
/goal_pose
/navigation_status
```

当前阶段：

```text
不接 ROS2
不生成地图
不伪造机器人位置
只保留函数接口和注释
```

接口示例：

```python
def publish_cmd_vel(vx: float, wz: float):
    pass

def send_navigation_goal(x: float, y: float, theta: float):
    pass

def get_slam_status():
    return None

def get_map_data():
    return None
```

# 8. 给 Codex 的通用固定前缀

```text
你正在开发一个运行在 Raspberry Pi 4B + 10.1 寸横屏电容触摸屏上的 UGV 遥控终端 Web UI。

项目目标：
- 作为无人探查车的遥控器和状态监视终端
- 前端使用 HTML/CSS/JavaScript
- 后端使用 Python Flask 或 FastAPI
- 实时数据使用 WebSocket
- UI 风格为深色科技风、蓝色边框、青色高亮、卡片式布局
- 必须适合触摸屏操作，按钮不能太小
- 请保持模块化，不要把所有代码写在一个文件里
- 请在现有项目基础上修改，不要重写已有架构

重要限制：
- 不要生成假数据
- 不要创建 fake_data.py、mock_data.py、demo_data.py
- 不要生成随机状态、随机日志、随机地图、随机雷达点
- 没有真实数据时，前端显示“未连接”“等待数据”或“暂无数据”
- 所有动态数据必须来自后端接口或 WebSocket
```
